v1.0.0 Node.js ≥ 18 Express 4.x JWT Bearer Auth SQLite

📋 项目概述

本服务为智慧园区管理平台提供 RESTful 后端 API，基于 Express 4.x + SQLite + JWT 构建， 具备企业级的安全防护（CORS、Helmet、限流）、统一响应格式和优雅关闭机制。

当前已实现接口

  • POST /auth/login — 用户登录
  • POST /auth/register — 用户注册
  • GET /auth/userinfo — 获取当前用户信息 🔒
  • POST /auth/refresh — 刷新 Access Token
  • POST /auth/logout — 退出登录 🔒
  • PUT /auth/password — 修改密码 🔒
  • GET /auth/captcha — 获取图片验证码

🔧 基础信息

开发环境地址http://localhost:3000
接口前缀无（前端 vite proxy 把 /api/* 映射到 http://localhost:3000/*）
Content-Typeapplication/json; charset=utf-8
鉴权方式请求头 Authorization: Bearer <token>
Token 有效期7 天（access token）/ 30 天（refresh token）
全局限流每 IP 每 15 分钟 200 次；登录接口每 IP 每 15 分钟 20 次
健康检查GET /health

📦 统一响应格式

所有接口均返回相同的 JSON 包装结构，与前端 ApiResponse<T> 类型完全对齐。

成功响应示例
{ "code": 0, "success": true, "data": { ... }, "message": "success", "timestamp": 1711123456789 }
失败响应示例
{ "code": 401, "success": false, "data": null, "message": "用户名或密码错误", "timestamp": 1711123456789 }

⚠️ 错误码说明

HTTP 状态码 业务 code 含义
2000请求成功
400400请求参数错误（字段缺失或格式不合规）
401401未授权：Token 缺失、无效或已过期；用户名或密码错误
403403账号已被禁用或无访问权限
404404资源不存在
409409资源冲突（用户名/邮箱已被注册）
429429请求过于频繁，触发限流
500500服务器内部错误

🔑 认证模块（/auth）

POST /auth/login 用户登录
登录成功后，将 token 存入 localStorage，后续请求在 Header 中携带 Authorization: Bearer <token>。
⚡ 限流：每 IP 每 15 分钟最多 20 次登录请求。
Request Body（application/json）
字段类型必填校验规则说明
username string 必填 非空 用户名
password string 必填 非空，长度 ≥ 6 密码（明文，HTTPS 传输）
captcha可选 string — — 图片验证码（配合 /auth/captcha 使用）
Request 示例
{ "username": "admin", "password": "admin123" }
Response Data（登录成功，code=0）
字段类型说明
tokenstringAccess Token，有效期 7 天
refreshTokenstringRefresh Token，有效期 30 天
expiresnumberAccess Token 过期时间戳（毫秒）
userInfoUserInfo见下方 UserInfo 数据结构
Response 示例
{ "code": 0, "success": true, "message": "登录成功", "data": { "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", "expires": 1712467200000, "userInfo": { "id": "a1b2c3d4-...", "username": "admin", "nickname": "管理员", "email": null, "phone": null, "avatar": null, "roles": ["admin", "user"], "permissions": ["*"], "createTime": "2026-03-23T09:00:00.000Z" } }, "timestamp": 1711123456789 }
可能的错误
401用户名或密码错误
403账号已被禁用
400参数校验失败（用户名/密码为空或密码过短）
429登录过于频繁，触发限流
POST /auth/register 用户注册
注册成功后自动跳转登录页，无需服务端自动登录。
Request Body（application/json）
字段类型必填校验规则说明
username string 必填 非空，正则 /^[A-Za-z0-9_]{4,20}$/ 用户名（4-20 位字母、数字或下划线）
password string 必填 非空，长度 ≥ 8 密码（建议包含大写字母、数字、特殊字符）
confirmPassword string 必填 非空，必须与 password 一致 确认密码
email可选 string — 若填写，需符合邮箱格式且全局唯一 邮箱地址
phone可选 string — 若填写，需符合中国手机号格式 手机号
captcha可选 string — — 图片验证码
Request 示例
{ "username": "testUser01", "password": "MyPass@2026", "confirmPassword": "MyPass@2026", "email": "test@example.com" }
Response Data（注册成功，code=0）
字段类型说明
userIdstring新用户 ID（UUID）
Response 示例
{ "code": 0, "success": true, "message": "注册成功", "data": { "userId": "b2c3d4e5-1234-5678-abcd-ef0123456789" }, "timestamp": 1711123456789 }
可能的错误
400参数校验失败（字段格式不合规、两次密码不一致）
409用户名已被占用 / 该邮箱已被注册
GET /auth/userinfo 获取当前用户信息 🔒 需要登录
需在请求头携带 Authorization: Bearer <token>，否则返回 401。
Request Headers
AuthorizationstringBearer <access_token>
Response Data（UserInfo 结构）
字段类型必有说明
idstring✓用户 ID（UUID）
usernamestring✓用户名
nicknamestring|null—昵称
emailstring|null—邮箱
phonestring|null—手机号
avatarstring|null—头像 URL
rolesstring[]✓角色列表，如 ["admin","user"]
permissionsstring[]✓权限列表，如 ["*"] 或 ["dashboard:read"]
createTimestring—创建时间（ISO 8601）
Response 示例
{ "code": 0, "success": true, "data": { "id": "a1b2c3d4-...", "username": "admin", "nickname": "管理员", "email": null, "phone": null, "avatar": null, "roles": ["admin", "user"], "permissions": ["*"], "createTime": "2026-03-23T09:00:00.000Z" }, "message": "success", "timestamp": 1711123456789 }
POST /auth/refresh 刷新 Access Token
当 access token 过期（收到 401）后，用 refresh token 换取新的 access token，无需重新登录。
Request Body
字段类型必填说明
refreshTokenstring必填登录时获取的 Refresh Token
Response Data

同登录接口 Response Data（token、refreshToken、expires、userInfo）

可能的错误
401refreshToken 无效或已过期，请重新登录
403账号已被禁用
POST /auth/logout 退出登录 🔒 需要登录
无状态 JWT 模式下，服务端直接返回成功，前端负责清除本地 token 和用户状态。
Request Headers
AuthorizationstringBearer <access_token>
Response 示例
{ "code": 0, "success": true, "data": null, "message": "已退出登录", "timestamp": 1711123456789 }
PUT /auth/password 修改密码 🔒 需要登录
Request Headers
AuthorizationstringBearer <access_token>
Request Body
字段类型必填校验规则说明
oldPassword string 必填 非空 当前密码
newPassword string 必填 非空，长度 ≥ 6 新密码
可能的错误
400原密码错误 / 新密码格式不合规
401未登录或 Token 已过期
GET /auth/captcha 获取图片验证码
当前为占位实现，后续可集成 svg-captcha + Redis 缓存。
Response Data
字段类型说明
captchaKeystring验证码唯一 Key，提交登录/注册时一并传入
captchaImgstringBase64 编码的 SVG 图片，直接赋值给 <img src>

📐 数据结构参考

UserInfo（与前端 types/auth.ts 完全对齐）
// TypeScript 类型参考（src/types/auth.ts） interface UserInfo { id : string // 用户 ID username : string // 用户名 nickname ?: string // 昵称 email ?: string // 邮箱 phone ?: string // 手机号 avatar ?: string // 头像 URL roles : string[] // 角色列表 permissions : string[] // 权限列表 createTime ?: string // 创建时间（ISO 8601） } interface LoginResult { token : string // Access Token refreshToken ?: string // Refresh Token expires ?: number // 过期时间戳（毫秒） userInfo : UserInfo // 用户信息 }
